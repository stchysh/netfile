# 文件共享功能设计方案

## 一、功能概述

在现有文件传输能力基础上，新增文件共享层：已完成接收的文件可以对局域网/信令网络内的在线设备开放访问，其他设备可搜索、浏览并下载共享文件。

---

## 二、核心概念

- **共享文件**：本地接收完成（status = completed）的文件记录，默认对所有在线设备可见
- **排除共享**：对单个文件/文件夹关闭共享，其他设备看不到该条目
- **标签**：附加在共享文件上的字符串标记，默认标签为本机昵称，最多 10 个
- **备注**：对共享文件的文字说明，随共享列表一起对外暴露
- **共享浏览**：打开共享 Tab 后主动拉取所有在线设备的共享列表，本地聚合搜索
- **收藏夹**：收藏远端共享文件（无需立即下载），随时可发起下载

---

## 三、需求明细

### 3.1 共享开关（全局）

- Config 新增 `enable_sharing: bool`，默认 `true`
- 关闭后：本设备不响应其他设备的共享列表查询，等同于"对外隐藏所有文件"
- 设置界面增加开关项

### 3.2 单文件排除共享

- 传输记录（TransferHistory）里，每条接收完成的记录新增"不共享"切换按钮
- 排除状态持久化在本地 ShareStore（独立于传输记录，以 `record_id` 关联）
- 排除的文件在其他设备的共享列表中不出现

### 3.3 文件标签

- 每个共享文件可添加最多 10 个标签（字符串，去重，单标签最长 32 字符）
- 默认标签：接收完成时自动打上本机 `instance_name`（昵称）
- 传输记录界面提供标签编辑入口：点击展开标签列，可增删标签
- 标签持久化在 ShareStore，随共享列表对外暴露

### 3.4 文件备注

- 每个共享文件/文件夹可附加一段文字备注（最多 100 字）
- 备注内容随 `SharedFileInfo` 一起返回给查询方，在共享 Tab 中显示在文件名下方
- 传输记录和共享 Tab 均提供备注编辑入口

### 3.5 下载确认配置

- Config 新增 `sharing_require_confirm: bool`，默认 `false`（不需要确认，对方直接下载）
- 开启后：其他设备发起下载时，需要本机用户手动确认，才真正开始传输（复用现有 `require_confirmation` 机制，但仅针对共享下载请求）
- `SharedFileInfo` 中携带 `require_confirm: bool` 字段，查询方在共享 Tab 中对需确认的文件显示"需确认"标记，用户点击下载后界面提示"已发送下载请求，等待对方确认"

### 3.6 分享热度（基于 MD5）

- `ShareEntry` 存储文件的 MD5 值（接收完成后异步计算）
- 每个设备维护自己的 `download_count`（该文件被其他设备下载的次数）
- `SharedFileInfo` 中携带：
  - `local_download_count: u32`：本设备的下载次数
  - `file_md5: Option<String>`：文件 MD5
- 查询方在共享 Tab 中，可将同一 MD5 的文件（来自不同设备）聚合显示：
  - 显示"N 台设备共享，共被下载 X 次"
  - 可展开看各设备的单独下载数
- 共享 Tab 支持"按热度排序"（跨设备同 MD5 合并计数排序）

### 3.7 单设备共享浏览

- 点击在线设备列表中某个设备 → 可查看该设备的全部共享文件
- 支持文件名搜索（本地过滤）
- 支持按标签筛选（点击标签 chip 过滤）
- 每条文件显示：文件名、备注、大小、标签、热度、是否需要确认
- 点击共享文件 → 发起下载

### 3.8 全局共享 Tab（搜索与浏览）

位于传输文件 / 传输记录旁边，新增"共享"切卡。

#### 数据加载策略：主动拉取 + 本地搜索

**方案对比**

| 方案 | 优点 | 缺点 |
|------|------|------|
| 打开 Tab 时并行拉取所有在线设备的共享列表 | 搜索即时（本地过滤），体验最好 | 首次加载有延迟 |
| 输入关键词后逐设备广播查询 | 无首屏等待 | 搜索慢、结果分批到达、体验割裂 |
| 发现广播中携带共享摘要 | 零延迟 | UDP 包大小受限，共享文件多时不可行 |

**采用方案：打开 Tab 时并行拉取**

- 打开共享 Tab → 向所有在线设备并行发送 `ShareListRequest`
- 每个设备响应返回其共享文件列表（只含元数据，不含文件内容）
- 前端聚合所有设备结果，全部在本地内存中做搜索/标签过滤
- 同 MD5 的文件自动聚合，显示跨设备下载总数
- 结果按设备分组显示，也支持跨设备混合搜索
- 顶部显示加载状态（哪些设备已响应，哪些超时）
- 提供手动刷新按钮；切换 Tab 再回来时数据缓存 30 秒有效

#### 共享 Tab 功能

- 搜索框：实时过滤文件名（跨所有设备）
- 标签筛选：展示所有已收到的标签，点击过滤
- 设备折叠分组：可按设备折叠/展开
- 每条结果显示：文件名、备注、大小、来源设备、标签、热度、是否需确认
- 点击"下载"→ 发起传输（需确认的文件提示等待状态）
- 点击"收藏"→ 加入本地收藏夹，不立即下载

### 3.9 收藏夹

- 共享 Tab 每条文件右侧提供"收藏"按钮
- 收藏记录本地持久化（存储来源设备地址 + file_id + 文件名 + 标签等元数据）
- 收藏夹为独立视图（共享 Tab 内切换），展示全部收藏文件
- 对每条收藏提供"下载"和"取消收藏"操作
- 若来源设备已离线，下载按钮显示灰色并标注"设备离线"

## 四、数据结构设计

### 4.1 ShareEntry（本地持久化，per 文件）

```rust
struct ShareEntry {
    record_id: String,           // 对应 TransferRecord.id
    file_name: String,
    file_size: u64,
    save_path: String,
    file_md5: Option<String>,    // 异步计算，完成前为 None
    tags: Vec<String>,           // 最多 10 个
    remark: String,              // 备注，最多 100 字，默认空
    excluded: bool,              // true = 不共享
    download_count: u32,         // 被其他设备下载次数
    timestamp: u64,
}
```

持久化路径：`~/.netfile/data/share_entries.json`

### 4.2 BookmarkEntry（收藏夹，本地持久化）

```rust
struct BookmarkEntry {
    id: String,                  // 本地唯一 id
    file_id: String,             // 来源设备的 file_id
    file_name: String,
    file_size: u64,
    tags: Vec<String>,
    remark: String,
    source_instance_id: String,
    source_instance_name: String,
    source_transfer_addr: String,
    require_confirm: bool,
    bookmarked_at: u64,
}
```

持久化路径：`~/.netfile/data/share_bookmarks.json`

### 4.3 网络协议扩展

在现有协议中新增消息类型（复用 TCP 传输端口）：

```rust
// 请求共享列表
ShareListRequest {
    requester_instance_id: String,
}

// 响应共享列表
ShareListResponse {
    instance_name: String,
    require_confirm: bool,       // 该设备是否全局开启下载确认
    entries: Vec<SharedFileInfo>,
}

// 单条共享文件元数据（不含内容）
SharedFileInfo {
    file_id: String,
    file_name: String,
    file_size: u64,
    file_md5: Option<String>,
    tags: Vec<String>,
    remark: String,
    download_count: u32,
    require_confirm: bool,       // 该文件下载是否需要确认（继承设备全局配置）
    timestamp: u64,
}
```

下载共享文件时，复用现有 `send_file` 命令，目标地址为共享设备的 transfer_addr。

### 4.4 Config 变更

在 `TransferConfig` 中新增：

```rust
#[serde(default = "default_enable_sharing")]
pub enable_sharing: bool,           // 默认 true

#[serde(default)]
pub sharing_require_confirm: bool,  // 默认 false
```


## 五、后端模块设计

### 5.1 ShareStore

```
ShareStore
  ├── load_entries() → Vec<ShareEntry>
  ├── upsert_entry(entry) → Result
  ├── set_excluded(record_id, excluded) → Result
  ├── update_tags(record_id, tags) → Result
  ├── update_remark(record_id, remark) → Result
  ├── get_shared_entries() → Vec<ShareEntry>      // excluded=false 的
  ├── increment_download_count(record_id) → Result
  └── update_md5(record_id, md5) → Result
```

### 5.2 BookmarkStore

```
BookmarkStore
  ├── load_bookmarks() → Vec<BookmarkEntry>
  ├── add_bookmark(entry) → Result
  ├── remove_bookmark(id) → Result
  └── get_bookmarks() → Vec<BookmarkEntry>
```

### 5.3 TransferService 扩展

- 接收文件完成后，自动向 ShareStore 写入一条 ShareEntry（tags 默认填 instance_name，MD5 异步计算后回写）
- 新增 TCP 消息处理分支：收到 `ShareListRequest` → 读 ShareStore + Config → 返回 `ShareListResponse`
- 收到共享下载请求时，若 `sharing_require_confirm = true`，走 pending_confirmations 流程（与普通接收确认共用），否则直接开始传输
- 传输完成后 `increment_download_count`

### 5.4 新增 Tauri 命令

```rust
get_share_entries()                              // 返回本地所有共享条目
set_share_excluded(record_id, excluded)          // 切换排除状态
update_share_tags(record_id, tags)               // 更新标签
update_share_remark(record_id, remark)           // 更新备注
query_device_shares(transfer_addr)               // 查询单个设备共享列表
query_all_shares()                               // 并行查询所有在线设备
get_bookmarks()                                  // 获取收藏列表
add_bookmark(entry)                              // 添加收藏
remove_bookmark(id)                              // 取消收藏
```

## 七、有趣的扩展玩法建议

### 7.1 时效共享（Expiry）
共享文件设置过期时间（1小时 / 1天 / 永久），过期后自动排除共享，适合临时性分发。

### 7.2 一键转发共享
下载别人的文件后，可以点击"转发共享"，自动将该文件加入自己的共享列表并打上来源设备昵称标签，形成文件在局域网内的传播链路。

### 7.3 共享通知
设备 A 新增共享文件时，通过信令服务器推送"新共享通知"给在线好友，好友端收到后在共享 Tab 角标显示红点，点击可看到新增内容。适合"发布文件"场景。

## 八、实现优先级

| 优先级 | 功能 |
|--------|------|
| P0 | Config 开关（enable_sharing、sharing_require_confirm）、ShareStore、接收完成自动入库 |
| P0 | TransferHistory 标签编辑、备注编辑、排除共享按钮 |
| P1 | ShareListRequest/Response 协议（含 require_confirm、remark、MD5 字段） |
| P1 | query_device_shares / query_all_shares 命令 |
| P1 | 共享 Tab（ShareBrowser）、全量拉取 + 本地搜索、需确认标记 |
| P1 | 收藏夹（BookmarkStore + UI） |
| P2 | MD5 异步计算、热度统计、跨设备 MD5 聚合显示 |
| P3 | 时效共享、一键转发共享、共享通知 |
