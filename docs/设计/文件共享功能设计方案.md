# 文件共享功能设计方案

## 一、功能概述

在现有文件传输能力基础上，新增文件共享层：已完成接收的文件可以对局域网/信令网络内的在线设备开放访问，其他设备可搜索、浏览并下载共享文件。

---

## 二、核心概念

- **共享文件**：本地接收完成（status = completed）的文件记录，默认对所有在线设备可见
- **排除共享**：对单个文件/文件夹关闭共享，其他设备看不到该条目
- **标签**：附加在共享文件上的字符串标记，默认标签为本机昵称，最多 10 个
- **共享浏览**：打开共享 Tab 后主动拉取所有在线设备的共享列表，本地聚合搜索

---

## 三、需求明细

### 3.1 共享开关（全局）

- Config 新增 `enable_sharing: bool`，默认 `true`
- 关闭后：本设备不响应其他设备的共享列表查询，等同于"对外隐藏所有文件"
- 设置界面增加开关项

### 3.2 单文件排除共享

- 传输记录（TransferHistory）里，每条接收完成的记录新增"不共享"切换按钮
- 排除状态持久化在本地 ShareStore（独立于传输记录，以 `record_id` 关联）
- 排除的文件在其他设备的共享列表中不出现

### 3.3 文件标签

- 每个共享文件可添加最多 10 个标签（字符串，去重，单标签最长 32 字符）
- 默认标签：接收完成时自动打上本机 `instance_name`（昵称）
- 传输记录界面提供标签编辑入口：点击展开标签列，可增删标签
- 标签持久化在 ShareStore

### 3.4 单设备共享浏览

- 点击在线设备列表中某个设备 → 可查看该设备的全部共享文件
- 支持文件名搜索（本地过滤）
- 支持按标签筛选（点击标签 chip 过滤）
- 点击共享文件 → 发起下载（复用现有传输流程）

### 3.5 全局共享 Tab（搜索与浏览）

位于传输文件 / 传输记录旁边，新增"共享"切卡。

#### 数据加载策略：主动拉取 + 本地搜索

**方案对比**

| 方案 | 优点 | 缺点 |
|------|------|------|
| 打开 Tab 时并行拉取所有在线设备的共享列表 | 搜索即时（本地过滤），体验最好 | 首次加载有延迟 |
| 输入关键词后逐设备广播查询 | 无首屏等待 | 搜索慢、结果分批到达、体验割裂 |
| 发现广播中携带共享摘要 | 零延迟 | UDP 包大小受限，共享文件多时不可行 |

**采用方案：打开 Tab 时并行拉取**

- 打开共享 Tab → 向所有在线设备并行发送 `ShareListRequest`
- 每个设备响应返回其共享文件列表（只含元数据，不含文件内容）
- 前端聚合所有设备结果，全部在本地内存中做搜索/标签过滤
- 结果按设备分组显示，也支持跨设备混合搜索
- 顶部显示加载状态（哪些设备已响应，哪些超时）
- 提供手动刷新按钮；切换 Tab 再回来时数据缓存 30 秒有效

#### 共享 Tab 功能

- 搜索框：实时过滤文件名（跨所有设备）
- 标签筛选：展示所有已收到的标签，点击过滤
- 设备折叠分组：可按设备折叠/展开
- 每条结果显示：文件名、大小、来源设备、标签、时间
- 点击"下载"→ 发起传输

## 四、数据结构设计

### 4.1 ShareEntry（本地持久化，per 文件）

```rust
struct ShareEntry {
    record_id: String,       // 对应 TransferRecord.id
    file_name: String,       // 冗余存储，避免跨表查询
    file_size: u64,
    save_path: String,
    tags: Vec<String>,       // 最多 10 个
    excluded: bool,          // true = 不共享
    share_count: u32,        // 被其他设备下载次数
    timestamp: u64,          // 来自 TransferRecord.timestamp
}
```

持久化路径：`~/.netfile/data/share_entries.json`

### 4.2 网络协议扩展

在现有协议中新增两个消息类型（复用 TCP 传输端口）：

```rust
// 请求共享列表
ShareListRequest {
    requester_instance_id: String,
}

// 响应共享列表
ShareListResponse {
    instance_name: String,
    entries: Vec<SharedFileInfo>,
}

// 单条共享文件元数据（不含内容）
SharedFileInfo {
    file_id: String,         // 用于后续发起下载请求
    file_name: String,
    file_size: u64,
    tags: Vec<String>,
    timestamp: u64,
}
```

下载共享文件时，复用现有 `send_file` 命令，目标地址为共享设备的 transfer_addr。

### 4.3 Config 变更

在 `TransferConfig` 中新增：

```rust
#[serde(default = "default_enable_sharing")]
pub enable_sharing: bool,   // 默认 true
```

### 4.4 前端共享文件类型

```typescript
interface SharedFileInfo {
  file_id: string
  file_name: string
  file_size: number
  tags: string[]
  timestamp: number
}

interface DeviceShares {
  instance_id: string
  instance_name: string
  transfer_addr: string
  files: SharedFileInfo[]
  loaded: boolean   // 是否已加载完成
  error?: string    // 超时或连接失败
}
```

## 五、后端模块设计

### 5.1 ShareStore

```
ShareStore
  ├── load_entries() → Vec<ShareEntry>
  ├── upsert_entry(entry) → Result
  ├── set_excluded(record_id, excluded) → Result
  ├── update_tags(record_id, tags) → Result
  ├── get_shared_entries() → Vec<ShareEntry>   // excluded=false 的
  └── increment_share_count(record_id) → Result
```

### 5.2 TransferService 扩展

- 接收文件完成后，自动向 ShareStore 写入一条 ShareEntry（tags 默认填 instance_name）
- 新增 TCP 消息处理分支：收到 `ShareListRequest` → 读 ShareStore → 返回 `ShareListResponse`
- 新增 `query_device_shares(addr)` 异步方法：向目标地址发 ShareListRequest，返回结果

### 5.3 新增 Tauri 命令

```rust
get_share_entries()                              // 返回本地所有共享条目
set_share_excluded(record_id, excluded)          // 切换排除状态
update_share_tags(record_id, tags)               // 更新标签
query_device_shares(transfer_addr)               // 查询单个设备共享列表
query_all_shares()                               // 并行查询所有在线设备
```

## 六、前端模块设计

### 6.1 新增 Share Tab

在 TransferQueue / TransferHistory 旁新增"共享"Tab，组件：`ShareBrowser.tsx`

布局：
```
[传输文件] [传输记录] [共享]

  ...
```

### 6.2 TransferHistory 改动

  - 标签展示区（可点击编辑）
  - "不共享"切换按钮

## 七、有趣的扩展玩法建议

### 7.1 文件说明（Description）
共享时可附加一段简短描述（最多 100 字），在其他设备浏览时显示在文件名下方，适合分享"这是什么文件"的上下文。

### 7.2 分享热度
ShareStore 记录 `share_count`（被下载次数），共享 Tab 支持"按热度排序"，直观反映哪些文件被下载最多。

### 7.3 时效共享（Expiry）
标签可以配合有效期：共享文件设置过期时间（1小时 / 1天 / 永久），过期后自动排除共享，适合临时性分发。

### 7.4 一键转发共享
下载别人的文件后，可以点击"转发共享"，自动将该文件加入自己的共享列表并打上来源设备昵称标签，形成文件在局域网内的传播链路。

### 7.5 收藏夹
共享 Tab 内可以收藏某个文件（本地记录该设备地址 + file_id），即使该文件尚未下载，下次也能快速定位和重新下载，类似书签。

### 7.6 共享通知
设备 A 新增共享文件时，通过信令服务器推送"新共享通知"给在线好友，好友端收到后在共享 Tab 角标显示红点，点击可看到新增内容。适合"发布文件"场景。

## 八、实现优先级

|--------|------|
| P0 | Config 开关、ShareStore、接收完成自动入库、标签默认昵称 |
| P0 | TransferHistory 标签编辑、排除共享按钮 |
| P1 | ShareListRequest/Response 协议、query_device_shares 命令 |
| P1 | 共享 Tab（ShareBrowser）、全量拉取 + 本地搜索 |
| P2 | share_count 统计、热度排序 |
| P2 | 收藏夹、转发共享 |
| P3 | 时效共享、共享通知 |
