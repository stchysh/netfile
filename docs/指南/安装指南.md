# NetFile 安装指南

## 系统要求

- **操作系统**: Windows 10+, Linux, macOS
- **Rust**: 1.70+ (仅编译时需要)
- **网络**: 局域网或互联网连接

## 从源码编译

### 1. 安装 Rust

访问 [https://rustup.rs/](https://rustup.rs/) 安装 Rust 工具链。

### 2. 克隆仓库

```bash
git clone https://github.com/yourusername/netfile.git
cd netfile
```

### 3. 编译项目

```bash
# 编译 Release 版本
cargo build --release

# 编译后的可执行文件位于
# - Windows: target/release/netfile.exe
# - Linux/macOS: target/release/netfile
```

### 4. 安装到系统

```bash
# 安装到 ~/.cargo/bin/
cargo install --path crates/netfile-cli

# 或者手动复制
cp target/release/netfile /usr/local/bin/  # Linux/macOS
```

## 从预编译二进制安装

### Windows

1. 下载最新的 `netfile-windows-x64.zip`
2. 解压到任意目录
3. 将目录添加到系统 PATH 环境变量

### Linux

```bash
# 下载
wget https://github.com/yourusername/netfile/releases/latest/download/netfile-linux-x64.tar.gz

# 解压
tar -xzf netfile-linux-x64.tar.gz

# 安装
sudo mv netfile /usr/local/bin/
sudo chmod +x /usr/local/bin/netfile
```

### macOS

```bash
# 使用 Homebrew
brew install netfile

# 或手动安装
curl -L https://github.com/yourusername/netfile/releases/latest/download/netfile-macos-x64.tar.gz | tar xz
sudo mv netfile /usr/local/bin/
```

## 验证安装

```bash
# 查看版本
netfile --version

# 查看帮助
netfile --help
```

## 首次运行

首次运行时，NetFile 会自动创建配置文件：

```bash
# 启动服务（会自动生成配置）
netfile start

# 配置文件位置
# - Windows: C:\Users\<用户名>\.netfile\config.toml
# - Linux/macOS: ~/.netfile/config.toml
```

## 卸载

### 从源码安装的

```bash
cargo uninstall netfile
```

### 手动安装的

```bash
# Linux/macOS
sudo rm /usr/local/bin/netfile

# Windows
# 删除安装目录并从 PATH 中移除
```

### 清理配置文件

```bash
# Linux/macOS
rm -rf ~/.netfile

# Windows
rmdir /s %USERPROFILE%\.netfile
```

## 故障排除

### 端口被占用

如果默认端口被占用，可以修改配置文件：

```toml
[network]
discovery_port = 37020  # 修改为其他端口
transfer_port = 37050   # 修改为其他端口
```

### 防火墙问题

确保防火墙允许以下端口：
- UDP 37020-37040 (设备发现)
- TCP 37050+ (文件传输)

```bash
# Linux (ufw)
sudo ufw allow 37020:37040/udp
sudo ufw allow 37050/tcp

# Windows (PowerShell 管理员)
New-NetFirewallRule -DisplayName "NetFile Discovery" -Direction Inbound -Protocol UDP -LocalPort 37020-37040 -Action Allow
New-NetFirewallRule -DisplayName "NetFile Transfer" -Direction Inbound -Protocol TCP -LocalPort 37050 -Action Allow
```

### 权限问题

Linux/macOS 上如果遇到权限问题：

```bash
# 给予执行权限
chmod +x netfile

# 如果需要绑定低端口（<1024），使用 setcap
sudo setcap 'cap_net_bind_service=+ep' /usr/local/bin/netfile
```
