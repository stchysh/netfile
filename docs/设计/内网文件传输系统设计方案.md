# 内网文件传输系统设计方案

## 项目概述

一个支持内网环境下的文件/文件夹传输工具，具备设备发现、断点续传、NAT穿透等核心能力。支持 GUI 和 CLI 两种使用模式，可在同一台机器上运行多个实例。

## 核心功能

### 1. 设备发现
- **局域网广播**: 使用 UDP 广播或 mDNS 协议自动发现同网段设备
- **设备列表**: 实时显示在线设备（设备名、IP、状态）
- **心跳机制**: 定期发送心跳包维持在线状态，超时自动移除

### 2. 文件传输
- **单文件传输**: 支持任意大小文件传输
- **文件夹传输**: 递归扫描并传输整个目录结构
- **拖拽上传**: 支持拖拽文件/文件夹到界面任意位置快速发送
- **断点续传**:
  - 传输前计算文件哈希（SHA256）
  - 分块传输（默认 1MB/块）
  - 记录已传输块索引
  - 中断后从上次位置继续
- **传输队列**: 支持多文件排队传输
- **进度显示**:
  - 默认显示任务整体进度（总文件数、总大小、总速度）
  - 可展开查看任务内每个文件的详细进度
  - 实时显示传输速度、剩余时间、完成百分比

### 3. NAT 穿透
- **STUN 协议**: 获取公网 IP 和端口映射
- **免费 STUN 服务器**:
  - Google: `stun.l.google.com:19302`
  - Cloudflare: `stun.cloudflare.com:3478`
  - OpenRelay: `stun.relay.metered.ca:80`
- **打洞策略**:
  - 优先尝试直连（同局域网）
  - 失败后尝试 UDP 打洞
  - 打洞失败则提示用户无法连接

### 4. 安全机制
- **传输加密**: 使用 TLS 1.3 加密传输通道
- **身份验证**:
  - 设备首次连接需要确认授权
  - 可选密码保护
- **文件校验**: 传输完成后校验哈希确保完整性

### 5. 多实例支持
- **动态端口分配**: 每个实例自动选择可用端口，避免端口冲突
- **实例标识**: 每个实例生成唯一 ID，在设备发现时区分同机器的不同实例
- **独立配置**: 每个实例可使用独立的配置文件（通过 `--config` 参数指定）
- **进程隔离**: 各实例独立运行，互不干扰
- **实例命名**: 支持为实例设置不同名称（如 "工作账号"、"个人账号"）

### 6. 命令行支持
- **CLI 模式**: 提供完整的命令行界面，无需 GUI 即可使用所有功能
- **脚本集成**: 支持在脚本中自动化文件传输任务
- **远程操作**: 可通过 SSH 在无图形界面的服务器上使用

## 技术架构

### 技术栈选型
- **语言**: Rust（高性能、内存安全）
- **网络框架**: tokio（异步运行时）
- **传输协议**:
  - TCP（可靠传输）
  - UDP（设备发现、打洞）
- **加密**: rustls（TLS 实现）
- **序列化**: bincode 或 protobuf（高效二进制协议）
- **GUI**: GPUI（Zed 编辑器使用的高性能 Rust GUI 框架）
- **CLI**: clap（命令行参数解析）

### 核心模块

#### 设备发现模块
```rust
// 广播消息格式
struct DiscoveryMessage {
    device_id: String,      // 设备唯一标识
    instance_id: String,    // 实例唯一标识（支持同机器多实例）
    device_name: String,    // 设备名称
    instance_name: String,  // 实例名称（可选，如 "工作账号"）
    version: String,        // 协议版本
    port: u16,              // 监听端口
    timestamp: u64,         // 时间戳
}
```

#### 传输协议
```rust
// 传输请求
struct TransferRequest {
    file_id: String,        // 文件唯一标识
    file_name: String,      // 文件名
    file_size: u64,         // 文件大小
    file_hash: [u8; 32],    // SHA256 哈希
    chunk_size: u32,        // 分块大小
}

// 传输数据块
struct ChunkData {
    file_id: String,        // 文件标识
    chunk_index: u32,       // 块索引
    data: Vec<u8>,          // 数据内容
    checksum: u32,          // 块校验和
}

// 传输状态
struct TransferStatus {
    file_id: String,
    completed_chunks: Vec<u32>,  // 已完成块列表
    total_chunks: u32,
}
```

## 实现建议

### 1. 分阶段开发
- **阶段 1**: 局域网设备发现 + 基础文件传输
- **阶段 2**: 断点续传 + 文件夹传输
- **阶段 3**: NAT 穿透
- **阶段 4**: GUI + 用户体验优化

### 2. 性能优化
- **零拷贝**: 使用 `sendfile` 系统调用减少内存拷贝
- **并发传输**: 多文件并行传输（可配置并发数）
- **流式处理**: 边读边传，避免大文件占用内存
- **压缩传输**: 可选启用 zstd 压缩（适合文本文件）

### 3. 用户体验
- **拖拽上传**:
  - 支持拖拽文件/文件夹到主界面任意位置
  - 支持拖拽到设备列表项直接发送给指定设备
  - 支持拖拽到发送确认窗口添加更多文件
- **任务进度管理**:
  - 默认显示任务整体进度（文件总数、总大小、总速度）
  - 点击任务可展开查看每个文件的详细进度
  - 支持暂停/恢复/取消单个任务
- **历史记录**: 保存传输历史和常用设备
- **通知提醒**: 传输完成后系统通知
- **暗色模式**: 支持明暗主题切换

### 4. 跨平台支持
- **Windows**: 使用 Windows API 获取设备信息
- **macOS**: 使用 Bonjour 服务发现
- **Linux**: 使用 Avahi 或原生 mDNS

### 5. 配置管理
```toml
# config.toml
[instance]
instance_id = ""               # 实例唯一标识（自动生成）
instance_name = "默认实例"     # 实例名称
device_name = "My-Computer"    # 设备名称

[network]
discovery_port = 0             # 发现端口（0 表示自动分配）
transfer_port = 0              # 传输端口（0 表示自动分配）
broadcast_interval = 5         # 广播间隔（秒）

[transfer]
chunk_size = 1048576           # 分块大小（1MB）
max_concurrent = 3             # 最大并发传输数
enable_compression = false     # 是否启用压缩

[security]
require_auth = true            # 是否需要授权
password = ""                  # 可选密码
```

### 6. 多实例实现要点

#### 端口管理
- 启动时自动扫描可用端口，避免冲突
- 配置文件中端口设为 0 表示自动分配
- 启动后将实际使用的端口写入运行时状态文件

#### 实例识别
- 每个实例生成唯一的 `instance_id`（UUID）
- 设备发现时同时广播 `device_id` 和 `instance_id`
- 设备列表显示格式：`设备名 - 实例名 (IP:端口)`

#### 配置隔离
- 支持 `--config` 参数指定配置文件路径
- 默认配置路径：`~/.netfile/config.toml`
- 多实例可使用：`~/.netfile/instance1.toml`、`~/.netfile/instance2.toml`

#### 数据隔离
- 每个实例使用独立的数据目录
- 传输历史、设备列表、断点续传状态分别存储
- 数据目录结构：
  ```
  ~/.netfile/
    ├── config.toml           # 默认配置
    ├── instance1.toml        # 实例1配置
    ├── instance2.toml        # 实例2配置
    ├── data/                 # 默认实例数据
    │   ├── devices.json
    │   ├── history.json
    │   └── transfers/
    ├── data-instance1/       # 实例1数据
    └── data-instance2/       # 实例2数据
  ```

## 命令行接口设计

### 基本命令

#### 启动服务
```bash
# GUI 模式（默认）
netfile

# CLI 模式（无图形界面）
netfile --no-gui

# 指定配置文件
netfile --config ~/.netfile/work.toml

# 指定实例名称
netfile --name "工作账号"

# 后台运行（守护进程）
netfile daemon --name "后台实例"
```

#### 设备管理
```bash
# 列出在线设备
netfile devices list

# 显示设备详情
netfile devices info <device-id>

# 刷新设备列表
netfile devices refresh
```

#### 文件传输
```bash
# 发送文件到指定设备
netfile send <device-id> <file-path>

# 发送多个文件
netfile send <device-id> file1.txt file2.pdf folder/

# 发送文件夹
netfile send <device-id> --recursive ./project/

# 启用压缩
netfile send <device-id> --compress file.txt

# 指定保存路径（接收方）
netfile send <device-id> --remote-path /tmp/ file.txt
```

#### 传输管理
```bash
# 查看传输队列
netfile transfers list

# 查看传输详情
netfile transfers info <transfer-id>

# 暂停传输
netfile transfers pause <transfer-id>

# 恢复传输
netfile transfers resume <transfer-id>

# 取消传输
netfile transfers cancel <transfer-id>

# 查看传输历史
netfile history
```

#### 配置管理
```bash
# 查看当前配置
netfile config show

# 设置配置项
netfile config set instance.name "我的实例"
netfile config set network.discovery_port 37020

# 重置配置
netfile config reset

# 导出配置
netfile config export > backup.toml

# 导入配置
netfile config import backup.toml
```

### 脚本示例

#### 批量传输脚本
```bash
#!/bin/bash
# 启动后台实例
netfile daemon --name "backup-service" --config backup.toml &

# 等待服务启动
sleep 2

# 查找目标设备
DEVICE_ID=$(netfile devices list --format json | jq -r '.[] | select(.name=="BackupServer") | .id')

# 发送备份文件
netfile send $DEVICE_ID --recursive /data/backup/

# 等待传输完成
netfile transfers wait --all

# 停止实例
netfile instances stop backup-service
```

## 技术难点

### 1. NAT 穿透成功率
- 不同 NAT 类型穿透难度不同
- 对称型 NAT 几乎无法打洞
- 打洞失败时需要明确提示用户

### 2. 断点续传可靠性
- 需要持久化传输状态
- 处理文件被修改的情况
- 清理未完成的临时文件

### 3. 大文件传输稳定性
- 内存占用控制
- 网络波动处理
- 超时重试机制

### 4. 跨平台兼容性
- 文件路径分隔符差异
- 文件权限处理
- 特殊字符编码

### 5. 多实例端口冲突
- 需要可靠的端口扫描机制
- 处理端口被其他程序占用的情况
- 实例间通信避免干扰
